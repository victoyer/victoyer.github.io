正则表达式 — 编程元素中的深闺怨妇

下面是使用正则需要用到的一些表示字符

匹配字符

字符 功能
. 匹配任意1个字符（除了\n）
[ ] 匹配[ ]中列举的字符
\d 匹配数字，即0-9
\D 匹配非数字，即不是数字
\s 匹配空白，即 空格，tab键
\S 匹配非空白
\w 匹配单词字符，即a-z、A-Z、0-9、_
\W 匹配非单词字符

数量表示符

字符 功能
* 匹配前一个字符出现0次或者无限次，即可有可无
+ 匹配前一个字符出现1次或者无限次，即至少有1次
? 匹配前一个字符出现1次或者0次，即要么有1次，要么没有
{m} 匹配前一个字符出现m次
{m,} 匹配前一个字符至少出现m次
{m,n} 匹配前一个字符出现从m到n次

边界表示符

字符 功能
^ 匹配字符串开头
$ 匹配字符串结尾
\b 匹配一个单词的边界
\B 匹配非单词边界

分组表示符

字符 功能
\ 匹配左右任意一个表达式
(ab) 将括号中字符作为一个分组
\num 引用分组num匹配到的字符串
(?P) 分组起别名
(?P=name) 引用别名为name分组匹配到的字符串

re模块的高级用法

上面是一些正则表达式基本会用到的字符，下面我们来看看Python的正则表达式一些高级的用法

————————-我是分割线——————————

1、search:搜索字符串中第一次出现正则表达式的模式 成功返回匹配对象 否则返回None
2、findll:查找字符串所有与正则表达式匹配的模式 返回字符串的列表
3、sub:用指定的字符串替换原字符串中与正则表达式匹配的模式 可以用count指定替换的次数
4、split 根据匹配进行切割字符串，并返回一个列表

像上面这么吊的还有好多。我都放在下面，有兴趣自己去试试吧

方法 说明
compile(pattern, flags=0) 编译正则表达式返回正则表达式对象
match(pattern, string, flags=0) 用正则表达式匹配字符串 成功返回匹配对象 否则返回None
search(pattern, string, flags=0) 搜索字符串中第一次出现正则表达式的模式 成功返回匹配对象 否则返回None
split(pattern, string, maxsplit=0, flags=0) 用正则表达式指定的模式分隔符拆分字符串 返回列表
sub(pattern, repl, string, count=0, flags=0) 用指定的字符串替换原字符串中与正则表达式匹配的模式 可以用count指定替换的次数
fullmatch(pattern, string, flags=0) match函数的完全匹配（从字符串开头到结尾）版本
findall(pattern, string, flags=0) 查找字符串所有与正则表达式匹配的模式 返回字符串的列表
finditer(pattern, string, flags=0) 查找字符串所有与正则表达式匹配的模式 返回一个迭代器
purge() 清除隐式编译的正则表达式的缓存
re.I / re.IGNORECASE 忽略大小写匹配标记
re.M / re.MULTILINE 多行匹配标记

贪婪匹配和非贪婪匹配

这个贪婪是在Python的正则匹配数量的默认词（大多数语言也是默认贪婪）意思是尽可能多的匹配字符

非贪婪匹配与之相反，是尽可能少的匹配字符

非贪婪匹配

在正则式中的通配字都会尽量匹配最长的字符串，所以在 * + ？ {m,n} 的后面加上 ？号可以使贪婪匹配变成非贪婪匹配

最后说一下注意点：

1、\w不只是匹配字母、数字和下划线，还有中文汉字以及平假名片假名什么的

2、split可以安照指定模式切割字符串并且返回一个list。但是最好少用，天知道它会给你 个啥东西，亦或者是我用的不得其法吧

3、 和 + 去匹配字符串的时候。如果是 会返回 “” (同？)，因为它们默认可以为0，这个0其实就是 “” ，如果没有匹配到的话返回的就是 “” ；而 + 是必须有一次，所以没有的话会返回None
